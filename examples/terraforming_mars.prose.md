# 테라포밍 마즈 — 웹 인터페이스 기획 입력 예제 (D32)

`ludoforge web`의 **① 기획 입력** 단계에 그대로 붙여 넣는 예제다. 카드 200여 장의 고유
메커니즘은 범위 밖(D32 비목표)이고, TM에서 검증 가능한 **수치·경제 서브시스템**(글로벌
파라미터·TR·세대 소득·표준 프로젝트)만 산문으로 쓴다. 참조 번역(정답 예시)은
[`terraforming_mars.lf`](terraforming_mars.lf) — LLM 번역 결과가 이와 구조적으로
같아야 하고, 다르면 규칙서 미리보기(사람 게이트)에서 걸러낸다.

## 산문 (기획 입력 칸)

```
화성 지구화 게임의 핵심 수치 시스템이다.

글로벌 파라미터는 세 가지다: 온도, 산소, 바다. 온도는 -30°C에서 +8°C까지 2°C
단위로 총 19단계 올릴 수 있다(단계 0~19로 표현). 산소는 0%에서 14%까지 14단계,
바다 타일은 0개에서 9개까지 놓을 수 있다. 파라미터는 내려가는 일이 없다.

테라포밍 등급(TR)은 20에서 시작하고, 어떤 파라미터든 1단계 올릴 때마다 TR이 1
오른다. 따라서 TR의 이론 최대는 62다(20+19+14+9).

게임은 세대 단위로 진행된다. 매 세대 플레이어는 TR과 같은 금액의 메가크레딧(MC)
수입을 받고, 표준 프로젝트를 최대 1개 수행하거나 아무것도 하지 않고 저축한다.
표준 프로젝트의 종류와 비용은 첨부한 시트(cost)와 같고, 각 프로젝트는 해당
파라미터를 1단계 올린다. 비용은 수입을 받은 뒤의 잔고에서 지불하며, 잔고가
부족하면 수행할 수 없다. 시작 잔고는 40 MC다.

플레이어 성향은 시트(policy)의 상대 가중치를 따른다(온도 우선). 저축의 상대
가중치는 1이다. 파라미터별 단계 상한은 시트(cap)와 같다.

세 파라미터가 모두 최대가 되면 게임이 즉시 끝난다.

검증하고 싶은 것:
- 지구화 완주(세 파라미터 모두 최대)가 실제로 가능해야 한다.
- TR은 절대 20 아래로 내려가거나 62를 넘으면 안 된다.
- 잔고는 음수가 될 수 없어야 한다.
- 게임이 막다른 상태(아무것도 못 하는 상태)에 빠지면 안 된다.
- 게임 길이(세대 수)와 종료 시점 잔고의 분포가 궁금하다.
```

## 시트 (CSV — 표 3장, LLM을 거치지 않고 그대로 `table` 절이 된다)

**`cap`** — 파라미터 단계 상한

```csv
param,steps
temp,19
oxy,14
ocean,9
```

**`cost`** — 표준 프로젝트 비용(MC)

```csv
param,mc
temp,14
oxy,23
ocean,18
```

**`policy`** — 플레이어 성향(상대 가중치)

```csv
param,weight
temp,4
oxy,2
ocean,3
```

## 기대 결과 (참조 번역 실측, 2026-07-07)

- `check`: 정적 모순 없음.
- `bmc --k 10`: TR 하한/상한·잔고 비음수 **k-귀납 무한 지평 증명**, 데드락 없음 증명.
  완주 도달성은 k=10에선 미확인(정직 보고) — `--k 45`에서 깊이 43 증인 경로(~5분).
- `sim -H 300 -n 20000`: 완주율 100%(이 정책 하 · Pmax 아님 라벨), 게임 길이 평균
  ~49.1세대(p5 45 · p95 55), 종료 잔고 평균 ~1,318 MC(p5 ~1,094 · p95 ~1,621).

이 예제가 보여주는 것: 도메인을 좁힌 뒤에도(D32) 테라포밍 마즈급 보드게임의 **경제
골격**은 산문+시트 입력 → 검증·추정까지 완주할 수 있다 — 못 담는 것은 카드별 고유
메커니즘이지, 수치 시스템이 아니다.
